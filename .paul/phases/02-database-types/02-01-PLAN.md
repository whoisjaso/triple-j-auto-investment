<objective>
## Goal
Define the Supabase database schema for v0.1 core entities (vehicles, leads), create matching TypeScript types, build typed query helpers, and provide seed data for development.

## Purpose
The data layer is the backbone of the entire application. Inventory display (Phase 5), lead capture (Phase 6), and admin (Phase 7) all depend on having well-typed database access. Getting this right now prevents type drift and rework later.

## Output
- SQL schema file for Supabase (vehicles + leads tables with RLS)
- TypeScript types matching the schema
- Typed query helpers for server/client components
- Seed data SQL for development (5-8 sample vehicles, 3 sample leads)
</objective>

<task type="auto">
  <name>Task 1: Create SQL schema and TypeScript types</name>
  <files>supabase/schema.sql, src/types/database.ts</files>
  <action>
    1. Create `supabase/schema.sql` with:

    **vehicles table:**
    - id: uuid primary key (gen_random_uuid())
    - make: text not null
    - model: text not null
    - year: integer not null
    - price: numeric(10,2) not null
    - mileage: integer not null
    - vin: text not null unique
    - status: text not null default 'Available' (check constraint: Available, Pending, Sold)
    - description: text
    - image_url: text
    - gallery: text[] (array of image URLs)
    - slug: text not null unique
    - body_style: text
    - exterior_color: text
    - interior_color: text
    - transmission: text
    - drivetrain: text
    - engine: text
    - fuel_type: text
    - date_added: timestamptz default now()
    - created_at: timestamptz default now()
    - updated_at: timestamptz default now()

    **leads table:**
    - id: uuid primary key (gen_random_uuid())
    - name: text not null
    - email: text
    - phone: text not null
    - message: text
    - vehicle_id: uuid references vehicles(id) on delete set null
    - source: text not null default 'contact_form' (check: contact_form, financing_inquiry, vehicle_inquiry, schedule_visit)
    - status: text not null default 'New' (check: New, Contacted, Closed)
    - created_at: timestamptz default now()

    **RLS policies:**
    - Enable RLS on both tables
    - vehicles: allow SELECT for anon (public can browse inventory)
    - leads: allow INSERT for anon (public can submit leads)
    - Both: allow ALL for authenticated (admin access)

    **Indexes:**
    - vehicles: index on status, slug
    - leads: index on status, created_at

    **Updated_at trigger:**
    - Create a trigger function that auto-updates updated_at on vehicles

    2. Create `src/types/database.ts` with:
    - VehicleStatus enum: 'Available' | 'Pending' | 'Sold'
    - LeadStatus enum: 'New' | 'Contacted' | 'Closed'
    - LeadSource type: 'contact_form' | 'financing_inquiry' | 'vehicle_inquiry' | 'schedule_visit'
    - Vehicle interface matching vehicles table columns (camelCase)
    - Lead interface matching leads table columns (camelCase)
    - VehicleInsert type (Omit Vehicle of id, created_at, updated_at, date_added)
    - LeadInsert type (Omit Lead of id, created_at)
    - VehicleFilters type for query params: { status?, minPrice?, maxPrice?, minYear?, maxYear?, make?, search? }

    Use snake_case for database column names in SQL, camelCase for TypeScript interfaces.
    The query helpers will handle the mapping.

    Avoid: Do NOT include rental, registration, plate, insurance, or behavioral tracking fields -- those are v0.2+
    Avoid: Do NOT include cost/financial fields (cost, costTowing, etc.) -- admin-only, Phase 7
  </action>
  <verify>Run `npx tsc --noEmit` to confirm types compile without errors</verify>
  <done>AC-1 and AC-2 satisfied: SQL schema and TypeScript types defined and aligned</done>
</task>

<task type="auto">
  <name>Task 2: Create typed query helpers and seed data</name>
  <files>src/lib/supabase/queries/vehicles.ts, src/lib/supabase/queries/leads.ts, supabase/seed.sql</files>
  <action>
    1. Create `src/lib/supabase/queries/vehicles.ts`:
    - Import SupabaseClient type and Vehicle/VehicleFilters types
    - getVehicles(client, filters?): fetch vehicles with optional filtering
      - Filter by status (default: 'Available')
      - Filter by make, minPrice, maxPrice, minYear, maxYear
      - Search by make+model text match
      - Order by date_added desc (newest first)
      - Return Vehicle[]
    - getVehicleBySlug(client, slug): fetch single vehicle by slug
      - Return Vehicle | null
    - getFeaturedVehicles(client, limit = 6): fetch latest available vehicles
      - Filter status = 'Available', order by date_added desc, limit
      - Return Vehicle[]
    - Note: Supabase returns snake_case, so either use .select() column aliases
      or map snake_case to camelCase. Prefer: keep snake_case in DB types and
      use a simple mapVehicle() helper to convert to camelCase Vehicle interface.

    2. Create `src/lib/supabase/queries/leads.ts`:
    - createLead(client, lead: LeadInsert): insert a new lead
      - Return the created Lead
    - Note: Input uses camelCase, convert to snake_case for insert

    3. Create `supabase/seed.sql`:
    - 6 realistic vehicles for a Houston BHPH dealership:
      - Mix of makes: Toyota, Honda, Nissan, Ford, Chevrolet, Hyundai
      - Years: 2015-2021
      - Prices: $3,500 - $7,500 range
      - Realistic mileage: 80K-150K
      - Real VIN format (17 chars, realistic patterns)
      - URL-friendly slugs: "2019-toyota-camry-se"
      - Descriptions written in warm, family-friendly dealership tone
      - Use placeholder image URLs (e.g., /images/vehicles/placeholder-1.jpg)
      - Include body_style, exterior_color, transmission, engine, fuel_type
    - 3 sample leads:
      - One general inquiry (contact_form)
      - One financing inquiry (financing_inquiry)
      - One vehicle-specific inquiry (vehicle_inquiry) referencing a vehicle

    Avoid: Do NOT create actual Supabase migrations -- just raw SQL files
    Avoid: Do NOT add admin query helpers yet -- Phase 7
  </action>
