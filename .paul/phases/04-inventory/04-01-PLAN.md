<objective>
## Goal
Build the `/inventory` page that displays vehicles in a filterable, sortable grid — with mock data fallback when Supabase isn't connected — matching the luxury aesthetic established in Phase 3.

## Purpose
The inventory page is the primary conversion surface. Facebook Marketplace traffic lands on vehicle detail pages (Plan 04-02), but the listings page is where users browse, compare, and qualify vehicles. It must load fast (server-rendered), feel luxurious, and get users to a vehicle detail page with minimal friction. The filter/sort system drives discovery for the $3-8K inventory range.

## Output
- `/inventory` route with server-rendered vehicle grid
- VehicleCard component with image, specs, price, and link to detail
- FilterBar with make, price range, year range, and text search
- SortSelect with 5 sort options (price, year, mileage, newest)
- Mock data (6 vehicles from seed.sql) for development without Supabase
- Sort parameter support added to existing getVehicles query
</objective>
