<objective>
## Goal
Set up next-intl i18n infrastructure with prefix-based routing (`/en/...`, `/es/...`), restructure the app directory under a `[locale]` segment, and create complete English/Spanish translation message files.

## Purpose
Triple J's primary customers are bilingual Hispanic families in Houston. A fully Spanish-language option removes friction for Spanish-dominant users arriving from Facebook Marketplace, directly supporting conversion from stranger to buyer. This is Phase 7 of 8 toward v0.1 launch.

## Output
- next-intl installed and configured
- All public routes under `/[locale]/` prefix (en default, es alternate)
- Middleware handles locale detection + admin route passthrough
- Complete en.json and es.json message files (~93 translation keys)
- Root layout restructured with NextIntlClientProvider
</objective>
