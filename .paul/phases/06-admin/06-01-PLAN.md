<objective>
## Goal
Build the admin authentication system, dashboard layout, and full inventory CRUD — giving Jason a protected admin panel to manage vehicles.

## Purpose
Jason needs to add, edit, and remove vehicles from the lot without touching code or a database console. The admin dashboard is the operational backbone of the dealership website. Every vehicle on the public site will flow through this admin panel. Without it, the site is static and Jason can't operate independently.

## Output
- `/admin/login` page with password-based authentication
- Next.js middleware protecting all `/admin/*` routes
- Admin layout with sidebar navigation (Dashboard, Inventory, Leads)
- `/admin/inventory` page listing all vehicles with status, actions
- `/admin/inventory/new` page with vehicle form + VIN auto-decode
- `/admin/inventory/[id]/edit` page for editing existing vehicles
- Delete vehicle with confirmation dialog
- Server actions for vehicle CRUD (Supabase when connected, mock fallback)
</objective>

    **Create `src/lib/actions/auth.ts`:**
    - "use server" directive
    - Export `loginAdmin(prevState: AuthState, formData: FormData): Promise<AuthState>`
      - Extract password from FormData
      - Compare against `process.env.ADMIN_PASSWORD` (fallback to "admin" if not set)
      - If match: create a session token using Node.js `crypto`:
        - `token = timestamp + "." + createHmac('sha256', secret).update(timestamp).digest('hex')`
        - `secret = process.env.ADMIN_SECRET || "dev-secret-triple-j"`
        - Set httpOnly cookie "admin-session" with token value (maxAge: 86400, path: "/", sameSite: "lax", secure in production)
      - Redirect to /admin
      - If mismatch: return { success: false, error: "Invalid password" }
    - Export `logoutAdmin(): Promise<void>`
      - Delete "admin-session" cookie
      - Redirect to /admin/login
    - Export `verifySession(cookieValue: string): boolean`
      - Split token by ".", verify HMAC matches
      - Check timestamp is within 24 hours
      - Return true/false

    **Create `src/middleware.ts`:**
    - Match paths: "/admin/:path*" (but NOT "/admin/login")
    - Check for "admin-session" cookie
    - If cookie missing or expired: redirect to /admin/login
    - If valid: allow request through (NextResponse.next())
    - Import verifySession logic inline (middleware runs in Edge runtime — use crypto.subtle for HMAC if needed, or keep it simple with a token check)

    **Important middleware consideration:** Next.js Edge middleware cannot use Node.js `crypto` module. Use `crypto.subtle` (Web Crypto API) for HMAC verification in middleware. Alternatively, use a simpler approach: sign the cookie with a known prefix that can be verified without crypto (e.g., base64-encoded JSON with a hash). Keep it pragmatic — the admin is a single user behind a password.

    **Simplest secure approach for middleware:**
    - Cookie value format: `{timestamp}.{hmac_hex}`
    - In middleware (Edge): use `crypto.subtle.importKey` + `crypto.subtle.sign` to verify HMAC
    - OR: just check the cookie exists and is non-empty for v0.1 (the login action already validates the password). The cookie itself is httpOnly so it can't be forged from JavaScript. This is acceptable security for a single-user admin on a dealership site.
    - Choose the pragmatic option: check cookie exists + verify format. Full HMAC verification in Edge is a nice-to-have.
