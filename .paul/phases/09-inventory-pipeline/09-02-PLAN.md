<objective>
## Goal
Connect the email parsers from Plan 09-01 to live Gmail data by building a Gmail REST API client with OAuth2 token refresh, a pipeline sync API endpoint that fetches → parses → creates/updates vehicles in Supabase, and a helper script for obtaining Google OAuth2 credentials.

## Purpose
This is the integration layer that makes the automated inventory pipeline real. Without Gmail connectivity, the parsers and schema from 09-01 are inert. This plan wires them together: one API call syncs Jason's triplejautoinvestment@gmail.com inbox into the vehicle pipeline, eliminating manual Google Sheet data entry.

## Output
- Gmail REST API client library (raw fetch, zero npm dependencies)
- `/api/pipeline/sync` POST endpoint (admin-authenticated, idempotent)
- OAuth2 token helper script for one-time credential setup
- Env vars: GMAIL_CLIENT_ID, GMAIL_CLIENT_SECRET, GMAIL_REFRESH_TOKEN
</objective>

## AC-1: Gmail API Client
```gherkin
Given valid GMAIL_CLIENT_ID, GMAIL_CLIENT_SECRET, and GMAIL_REFRESH_TOKEN env vars
When the Gmail client calls searchEmails() with a query string
Then it returns an array of email message IDs matching the query
And it auto-refreshes the access token when expired using the refresh token

Given a message ID
When getEmailContent() is called
Then it returns the email sender, subject, and decoded body (plain text or HTML)
And handles base64url-encoded content correctly
```

## AC-2: Pipeline Sync Endpoint
```gherkin
Given an authenticated admin request to POST /api/pipeline/sync
When the endpoint executes
Then it searches Gmail for emails from known pipeline senders (support@ove.com, protected@dealshield.com, do-not-reply@centraldispatch.com)
And classifies each email using classifyEmail()
And parses matching emails using the appropriate parser
And creates new vehicles from purchase confirmations (if VIN not already in DB)
And updates existing vehicles from guarantee and transport emails (matching by VIN)
And logs each processed email's Gmail message ID as source_email_id
And returns a JSON summary: { created: N, updated: N, skipped: N, errors: [] }

Given an email that was already processed (source_email_id exists in vehicle_events)
When the sync runs again
Then that email is skipped (idempotent — no duplicate vehicles or events)

Given an unauthenticated request to POST /api/pipeline/sync
When the request is received
Then it returns 401 Unauthorized
```

## AC-3: OAuth2 Token Helper
```gherkin
Given GMAIL_CLIENT_ID and GMAIL_CLIENT_SECRET are set
When the helper script runs
Then it prints a URL for the user to visit for Google OAuth consent
And accepts the authorization code from the user
And exchanges it for access + refresh tokens
And prints the GMAIL_REFRESH_TOKEN value to add to env vars
```

<task type="auto">
  <name>Task 1: Gmail REST API Client Library</name>
  <files>src/lib/gmail.ts</files>
  <action>
    Create `src/lib/gmail.ts` — a zero-dependency Gmail REST API client using raw fetch:

    1. Token management:
       - getAccessToken(): uses GMAIL_REFRESH_TOKEN + client credentials to get fresh access token
       - Cache token in module-level variable with expiry check
       - Token refresh URL: https://oauth2.googleapis.com/token
       - POST with grant_type=refresh_token, client_id, client_secret, refresh_token

    2. searchEmails(query: string, maxResults?: number): Promise<string[]>
       - GET https://gmail.googleapis.com/gmail/v1/users/me/messages?q={query}&maxResults={maxResults}
       - Returns array of message IDs
       - Default maxResults: 20

    3. getEmailContent(messageId: string): Promise<GmailMessage>
       - GET https://gmail.googleapis.com/gmail/v1/users/me/messages/{messageId}?format=full
       - Extract from headers: From, Subject, Date
       - Decode body: check parts for text/plain and text/html
       - Handle base64url encoding (replace - with +, _ with /, atob or Buffer.from)
       - Return: { id, sender, subject, date, bodyText, bodyHtml }

    4. Types:
       - GmailMessage: { id: string; sender: string; subject: string; date: string; bodyText: string; bodyHtml: string }
       - GmailTokenResponse (internal)

    5. Error handling:
       - If GMAIL_REFRESH_TOKEN not set, throw clear error
       - If token refresh fails, throw with message about re-authorizing
       - If API returns 401, attempt one token refresh retry

    Avoid: Using googleapis npm package (large, bundling issues with serverless).
    Avoid: Storing tokens in database — refresh token is an env var, access token is cached in memory.
    Avoid: Any write operations to Gmail (no marking as read, no deleting).
  </action>
  <verify>npx tsc --noEmit</verify>
  <done>AC-1 satisfied: Gmail client fetches emails with auto token refresh</done>

<task type="auto">
  <name>Task 2: Pipeline Sync API Endpoint</name>
  <files>src/app/api/pipeline/sync/route.ts</files>
  <action>
    Create `src/app/api/pipeline/sync/route.ts` — POST endpoint that orchestrates the full sync:

    1. Authentication:
       - Check for admin cookie (same HMAC verification as middleware)
       - Import ADMIN_SECRET from env, verify the admin_token cookie
       - Return 401 if not authenticated
       - Use crypto.subtle.importKey + crypto.subtle.verify for HMAC check (same as middleware.ts)

    2. Sync logic:
       - Build Gmail search query for pipeline senders:
         "from:support@ove.com OR from:protected@dealshield.com OR from:do-not-reply@centraldispatch.com"
       - Optional: accept `after` query param to limit time range (default: last 30 days)
         Add "after:{YYYY/MM/DD}" to Gmail query
       - Call searchEmails() to get message IDs
       - For each message ID:
         a. Check if source_email_id already exists in vehicle_events → skip if yes
         b. Call getEmailContent() to fetch full email
         c. Call processEmail(sender, subject, body) from email-parsers
         d. Based on result type:
            - "purchase": check if VIN exists in vehicles → if not, createVehicleFromPurchase()
            - "guarantee": updateVehicleFromGuarantee()
            - "transport_accepted"|"transport_picked_up"|"transport_delivered": updateVehicleFromTransport()
            - "high_bid": createVehicleFromPurchase() with status override to 'Bidding'
            - "unknown": skip
         e. Track results: created[], updated[], skipped[], errors[]

    3. Duplicate prevention:
       - Before processing, query vehicle_events WHERE source_email_id = messageId
       - If exists, add to skipped and continue
       - This makes the endpoint fully idempotent

    4. Response:
       - Return JSON: { success: true, created: N, updated: N, skipped: N, errors: string[], details: { created: [...vins], updated: [...vins] } }
       - On Gmail API error (no credentials): return 503 with helpful message

    5. Handle missing Gmail credentials gracefully:
       - If GMAIL_REFRESH_TOKEN not set, return 503 with { error: "Gmail not configured. Set GMAIL_CLIENT_ID, GMAIL_CLIENT_SECRET, GMAIL_REFRESH_TOKEN env vars." }

    Import from:
    - '@/lib/gmail' for searchEmails, getEmailContent
    - '@/lib/email-parsers' for processEmail
    - '@/lib/supabase/queries/pipeline' for createVehicleFromPurchase, updateVehicleFromGuarantee, updateVehicleFromTransport, getVehicleByVin
    - '@/lib/nhtsa' (indirectly via pipeline queries)

    Avoid: Modifying any existing API routes.
    Avoid: Running sync automatically on a timer (that's future work — for now admin triggers manually).
    Avoid: Writing to Gmail (no marking emails as read).
  </action>

    1. Read GMAIL_CLIENT_ID and GMAIL_CLIENT_SECRET from .env.local (or prompt)
    2. Print the Google OAuth2 authorization URL:
       https://accounts.google.com/o/oauth2/v2/auth?
         client_id={CLIENT_ID}&
         redirect_uri=http://localhost&

         scope=https://www.googleapis.com/auth/gmail.readonly&

       POST https://oauth2.googleapis.com/token

    6. Print the refresh_token value
    7. Print instructions: "Add this to .env.local and Vercel env vars:
       GMAIL_REFRESH_TOKEN={token}"

    Use only Node.js built-ins (readline, https/fetch).
    Add a comment at the top explaining when/why to run this script.

    Avoid: Auto-writing to .env.local (user should do this manually for security).
    Avoid: Storing access tokens (only the refresh token matters for env vars).
