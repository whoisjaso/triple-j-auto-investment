## Goal
Build the admin Pipeline dashboard page with a "Sync from Gmail" button, pipeline vehicles grouped by lifecycle stage, status advancement controls, and vehicle event timeline — giving Jason full visibility and control over the Manheim-to-website vehicle flow.

## Purpose
Plans 09-01 and 09-02 built the data layer and Gmail integration. Without a UI, Jason has no way to trigger syncs or manage the pipeline. This plan delivers the admin interface that replaces the Google Sheet workflow entirely — one click to sync emails, visual pipeline stages, and manual controls to advance vehicles toward listing.

## Output
- "Pipeline" nav item in admin sidebar (desktop + mobile)
- `/admin/pipeline` page with sync button, stage-grouped vehicle cards, status controls
- Client component for sync interaction + status advancement
- Server actions for advancing vehicle status
</objective>

## AC-1: Pipeline Navigation
```gherkin
Given the admin sidebar

```

## AC-2: Sync from Gmail
```gherkin
Given the admin pipeline page
When the user clicks the "Sync from Gmail" button
Then a loading state is shown during the sync
And the button calls POST /api/pipeline/sync
And on completion, a results summary is displayed: X created, Y updated, Z skipped
And any errors are shown
And the vehicle list refreshes to show newly created vehicles
```

## AC-3: Pipeline Stage View
```gherkin
Given vehicles exist in pipeline stages (Bidding, Purchased, In_Transit, Arrived, Inspection)
When the admin views /admin/pipeline
Then vehicles are displayed grouped by stage with stage headers and counts
And each vehicle card shows: year/make/model, VIN, purchase price, transport carrier, transport ETAs
And stages are ordered: Bidding → Purchased → In Transit → Arrived → Inspection
And empty stages show a subtle "No vehicles" message
And if no pipeline vehicles exist at all, a helpful empty state is shown
```

## AC-4: Status Advancement
```gherkin
Given a vehicle in a pipeline stage
When the admin clicks the "Advance" button on the vehicle card
Then the vehicle status advances to the next logical stage:
  Bidding → Purchased → In Transit → Arrived → Inspection → Available
And a vehicle_event is logged with type 'status_changed'
And the page refreshes to reflect the new status
And when advancing to "Available", the vehicle appears on the public inventory page
```

## AC-5: Pipeline Stats on Dashboard
```gherkin
Given vehicles exist in pipeline stages
When the admin views the main dashboard (/admin)
Then a "Pipeline" stat card shows the count of vehicles in pipeline stages
And it links to /admin/pipeline
```
