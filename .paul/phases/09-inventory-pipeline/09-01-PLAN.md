<objective>
## Goal
Expand the database schema to support the full vehicle lifecycle (BIDDING → PURCHASED → IN_TRANSIT → ARRIVED → INSPECTION → LISTED → PENDING → SOLD), add pipeline-specific columns for purchase/transport/guarantee data, create a vehicle_events audit trail table, and build email parser utilities that extract structured vehicle data from Manheim/OVE, DealShield, and Central Dispatch emails.

## Purpose
This is the data foundation for the entire Automated Inventory Pipeline. Without expanded schema and reliable email parsers, no automated vehicle creation or transport tracking is possible. The parsers convert machine-generated emails into structured records that flow directly into Supabase — eliminating Jason's manual Google Sheet workflow.

## Output
- Database migration SQL expanding vehicles table + creating vehicle_events table
- TypeScript types for pipeline statuses, events, and parsed email data
- Pipeline query functions (create from email, update status, log events)
- Pure email parser functions for OVE purchase confirmations, DealShield guarantees, and Central Dispatch transport notifications
- All parsers tested against known email format patterns from research
</objective>

  <files>src/lib/email-parsers/ove-parser.ts, src/lib/email-parsers/dealshield-parser.ts, src/lib/email-parsers/central-dispatch-parser.ts, src/lib/email-parsers/index.ts</files>

    Create `src/lib/email-parsers/` directory with pure parser functions:

    1. `ove-parser.ts` — parseOvePurchaseConfirmation(subject: string, body: string): ParsedPurchase | null
       - Extract from subject via regex: Year, Make, Model, Trim, VIN, Seller Name
         Pattern: /Manheim\.com Purchase Confirmation \((\d{4})\s+(.+?)\s+(\w+)\s+(.+?)\s+([A-HJ-NPR-Z0-9]{17})\s+(.+?)\)/
       - Extract from HTML body (machine-generated, consistent format):
         - Mileage: look for pattern like "167,898 mi" — regex /(\d[\d,]+)\s*mi/i
         - Colors: "Blue ext" → exterior_color, "BLK int" → interior_color
         - Pick-Up Location: city, state, zip pattern
         - Work Order Number, Stock Number: labeled fields
         - Purchase Amount, Buy Fee, Total Amount: dollar amounts
         - Facilitating Location (auction_location)
         - Purchase Date
       - Strip HTML tags for body parsing (simple regex: /<[^>]*>/g replace with spaces, normalize whitespace)
       - Return typed ParsedPurchase object or null if subject doesn't match

    2. `dealshield-parser.ts` — parseDealShieldGuarantee(subject: string, body: string): ParsedGuarantee | null
       - Subject match: /Purchase Confirmation: VIN: ([A-HJ-NPR-Z0-9]{17})/
       - Body is plain text with "Key: Value" format — parse line by line:
         - VIN, Year, Make, Model
         - Odometer (miles)
         - Vehicle Purchase Price (dollar amount)
         - Date Guaranteed, Guarantee Expiration (dates)
         - Guarantee Price (dollar amount)
         - Location
       - Return ParsedGuarantee or null

    3. `central-dispatch-parser.ts` — parseCentralDispatchNotification(subject: string, body: string): ParsedTransport | null
       - Subject match: /Load ID: (\d+) has been (ACCEPTED|PICKED UP|DELIVERED) by (.+)/
       - Normalize "PICKED UP" → "PICKED_UP" for TransportStatus
       - Body is plain text with labeled fields:
         - Pick Up Location, Delivery Location
         - Requested Pick Up/Delivery dates
         - Carrier Pick Up/Delivery ETA
         - Price (transport cost)
         - Number of Vehicles
         - Vehicle lines: Year Make Model + VIN (regex per line)
       - Return ParsedTransport or null

    4. `index.ts` — barrel export + unified processEmail(sender: string, subject: string, body: string) function:
       - Classify email by sender address (support@ove.com, protected@dealshield.com, do-not-reply@centraldispatch.com)
       - Route to appropriate parser
       - Return { type: 'purchase' | 'guarantee' | 'transport' | 'unknown', data: ParsedPurchase | ParsedGuarantee | ParsedTransport | null }

    Avoid: Any network calls — parsers are pure functions taking string input.
    Avoid: Complex HTML parsing libraries — these are machine-generated templates with consistent patterns, simple regex is sufficient.
    Avoid: Handling edge cases for human-written emails — all source emails are automated system notifications.

  <action>
    Create `src/lib/supabase/queries/pipeline.ts` with functions that bridge parsed email data → Supabase records:

    1. `createVehicleFromPurchase(client, parsed: ParsedPurchase): Promise<Vehicle>`
       - Call NHTSA decodeVin(parsed.vin) to auto-fill: engine, transmission, drivetrain, fuel_type, body_style
       - Merge NHTSA data with parsed email data (email data takes priority for: year, make, model, trim, mileage, colors)
       - Auto-generate slug: `${year}-${make}-${model}-${vin.slice(-6)}`.toLowerCase().replace(/\s+/g, '-')
       - Insert into vehicles with status='Purchased' and all pipeline fields
       - Log vehicle_event with type='purchased' and full parsed data as event_data
       - Return the created vehicle

    2. `updateVehicleFromGuarantee(client, parsed: ParsedGuarantee): Promise<Vehicle | null>`
       - Find vehicle by VIN
       - Update guarantee_expires_at, guarantee_price
       - Log vehicle_event with type='guarantee_confirmed'
       - Return updated vehicle or null if VIN not found

    3. `updateVehicleFromTransport(client, parsed: ParsedTransport): Promise<Vehicle[]>`
       - For each vehicle VIN in parsed.vehicles:
         - Find vehicle by VIN
         - Update transport fields: carrier, load_id, cost, ETAs
         - Update status: ACCEPTED/PICKED_UP → 'In_Transit', DELIVERED → 'Arrived'
         - Log vehicle_event with appropriate type
       - Return array of updated vehicles

    4. `getVehicleEvents(client, vehicleId: string): Promise<VehicleEvent[]>`
       - Query vehicle_events WHERE vehicle_id = vehicleId ORDER BY created_at DESC
       - Return mapped events

       - Query vehicles WHERE status NOT IN ('Available', 'Pending', 'Sold')
       - ORDER BY updated_at DESC
       - Returns all vehicles in pipeline stages (Bidding, Purchased, In_Transit, Arrived, Inspection)

    6. `advanceVehicleStatus(client, vehicleId: string, newStatus: PipelineStatus, notes?: string): Promise<Vehicle>`
       - Update vehicle status
       - Log vehicle_event with type='status_changed' and {from, to, notes} in event_data
       - Return updated vehicle
