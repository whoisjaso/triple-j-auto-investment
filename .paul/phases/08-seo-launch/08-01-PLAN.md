<objective>
## Goal
Add production-grade SEO infrastructure: dynamic html lang attribute, locale-specific metadata with Open Graph tags for every public page, JSON-LD structured data (AutoDealer + Vehicle schemas), auto-generated locale-aware sitemap, and robots.txt. Complete Phase 8 and close v0.1 milestone.

## Purpose
Phase 8 makes the site discoverable. Without SEO, the cinematic design and bilingual content are invisible to search engines. Google uses hreflang and structured data to serve the right language version to Houston's Hispanic community. Vehicle JSON-LD enables rich snippets in search results. The sitemap ensures all locale variants are indexed.

## Output
- Dynamic `<html lang>` matching the active locale
- Every public page has locale-specific `<title>`, `<meta description>`, Open Graph + Twitter Card tags
- JSON-LD `AutoDealer` schema on homepage, `Vehicle` schema on detail pages
- Auto-generated sitemap.xml with hreflang alternates for all locale/page combinations
- robots.txt allowing all crawlers with sitemap pointer
- Build passes with zero errors
</objective>

  <action>
    1. **AutoDealer JSON-LD (src/app/[locale]/layout.tsx):**
       - Add a `<script type="application/ld+json">` in the layout return (inside the NextIntlClientProvider wrapper)
       - Schema type: "AutoDealer" (extends LocalBusiness)
       - Include: @context, @type, name ("Triple J Auto Investment"), url, telephone, address (8774 Almeda Genoa Rd, Houston, TX 77075), openingHoursSpecification (Mon-Sat 9-19), priceRange ("$3,000 - $8,000"), image (logo URL)
       - Use `dangerouslySetInnerHTML` with JSON.stringify for the script content
       - This schema appears on ALL public pages (in the layout)

    2. **Vehicle JSON-LD (src/app/[locale]/inventory/[slug]/page.tsx):**
       - Add a `<script type="application/ld+json">` after the page content
       - Schema type: "Vehicle" (or "Car" with "Product" combined)
       - Include: @context, @type, name (year make model), brand (make), model, vehicleModelDate (year), mileageFromOdometer (with QuantitativeValue), vehicleIdentificationNumber (VIN), color (exteriorColor), vehicleTransmission, driveWheelConfiguration (drivetrain), fuelType, bodyType
       - Include offers: @type "Offer", price, priceCurrency "USD", availability "InStock", seller (reference to AutoDealer)
       - Only include fields that have values (skip null fields)

    3. **Sitemap (src/app/sitemap.ts):**
       - Export default async function `sitemap()` returning `MetadataSitemap`
       - Static pages: /, /inventory, /financing, /contact, /vin-decoder
       - For each static page, create entries for both /en/page and /es/page
       - Each entry should have `alternates.languages` with { en: "/en/page", es: "/es/page" }
       - Dynamic pages: fetch vehicle slugs (mock or Supabase) and create entries for each vehicle detail page in both locales
       - Set appropriate `lastModified` dates (use current date for static, vehicle.updatedAt for dynamic)
       - Set `changeFrequency` and `priority` (homepage: daily/1.0, inventory: daily/0.9, vehicles: weekly/0.8, others: monthly/0.5)

    4. **Robots.txt (src/app/robots.ts):**
       - Export default function returning `MetadataRoute.Robots`
       - Allow all user agents for all paths
       - Disallow /admin/* (admin routes should not be indexed)
       - Include sitemap URL pointing to /sitemap.xml
       - Use NEXT_PUBLIC_BASE_URL env var if available, fallback to relative /sitemap.xml

    Avoid:
    - Do NOT add structured data to admin pages
    - Do NOT modify vehicle data or queries — only read existing data for schema generation
    - Do NOT add sitemap entries for admin routes
    - Keep JSON-LD minimal and accurate — only include fields that have real values
