<objective>
## Goal
Build a Business Intelligence analytics dashboard at /admin/analytics that gives Jason at-a-glance insight into vehicle profitability, inventory performance, lead conversion rates, and source attribution — enabling data-driven decisions on what to buy and how to sell.

## Purpose
Jason currently has no visibility into which vehicles are most profitable, how long inventory sits, or which lead sources convert best. This dashboard transforms raw data already in Supabase (purchase costs, sale prices, repair costs, lead statuses, sources) into actionable business intelligence. Every metric answers a question Jason asks: "Am I making money?", "What should I buy more of?", "Where are my best leads coming from?"

## Output
- `/admin/analytics` — Full BI dashboard page with KPIs, profitability table, lead funnel, source attribution, and vehicle type performance
- Analytics queries in `src/lib/supabase/queries/analytics.ts`
- Mock data fallback in `src/lib/mock-analytics.ts`
- "Analytics" link in admin sidebar navigation
- Quick action card on admin dashboard
</objective>

  <action>
    1. Create src/lib/supabase/queries/analytics.ts with these query functions:

       a. `getVehicleProfitability(client)` — fetch all sold vehicles with financial fields:
          - SELECT id, year, make, model, purchase_price, buy_fee, total_cost, mechanical_cost, cosmetic_cost, other_costs, transport_cost, sale_price, selling_fees, net_profit, date_added, date_listed, date_sold, days_in_stock
          - WHERE status = 'Sold'
          - ORDER BY net_profit DESC NULLS LAST
          - Return typed array with computed fields:
            - repairCost = (mechanicalCost ?? 0) + (cosmeticCost ?? 0) + (otherCosts ?? 0)
            - totalInvested = (totalCost ?? 0) + repairCost + (transportCost ?? 0)
            - profit = (salePrice ?? 0) - totalInvested
            - marginPct = salePrice > 0 ? ((profit / salePrice) * 100) : 0

       b. `getKpiSummary(client)` — aggregate KPIs:
          - Total profit: SUM of net_profit from sold vehicles (or compute from sale_price - total_cost)
          - Avg profit per vehicle: total profit / sold count
          - Avg days on lot: AVG of days_in_stock for sold vehicles
          - Total sold: COUNT where status = 'Sold'
          - Lead conversion rate: (leads with status='Sold') / (total leads) * 100
          - Use parallel Promise.all for efficiency

       c. `getLeadFunnelData(client)` — lead counts per stage:
          - Reuse getLeadCountsByStatus pattern from leads.ts
          - Return ordered array: [{stage, count, color}] matching 7-stage pipeline
          - Include total leads count for funnel proportions

       d. `getLeadSourceAttribution(client)` — per-source conversion:
          - For each source in ('contact_form', 'financing_inquiry', 'vehicle_inquiry', 'schedule_visit'):
            - COUNT total leads with that source
            - COUNT leads with that source AND status = 'Sold'
            - Compute conversion rate
          - Return sorted by conversion rate DESC

       e. `getVehicleTypePerformance(client)` — make/model ranking:
          - GROUP BY make, model WHERE status = 'Sold' AND sale_price IS NOT NULL
          - For each group: COUNT, AVG(net_profit), AVG(days_in_stock)
          - ORDER BY avg profit DESC
          - LIMIT 10 (top performers)
