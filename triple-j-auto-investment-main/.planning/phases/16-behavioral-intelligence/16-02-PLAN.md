<objective>
Add conversion attribution to every lead capture -- every form submission records where the visitor came from (page, referrer, UTM params, device, session).

Purpose: Without attribution, Triple J cannot know which marketing channels drive leads. This plan captures source data on the existing leads table so the admin can see which pages, campaigns, and devices generate the most conversions. The key design goal is that attribution is captured automatically at the addLead layer -- zero changes needed to the 6+ existing form components.

Output: attributionService.ts (UTM capture + attribution assembly), updated vehicleLeadService.ts and leads.ts to include attribution fields.
</objective>

**attributionService.ts:**
Create a new service with three exports:

1. `captureInitialUtm()`: Read `window.location.search` via URLSearchParams. If `utm_source` exists, save `{ utm_source, utm_medium, utm_campaign }` to `sessionStorage` under key `tj_utm`. This should be called ONCE on app initialization (in App.tsx or StoreProvider). Only captures on initial page load -- SPA navigation won't have UTM params in the URL.

2. `getDeviceType()`: Return 'mobile' | 'tablet' | 'desktop' based on `window.innerWidth` (< 768 = mobile, < 1024 = tablet, else desktop). Note: If Plan 01 also defines getDeviceType in trackingService.ts, that's fine -- this is a tiny utility function, and having it in both avoids a cross-service dependency. Alternatively, import from trackingService if it exists.

3. `captureAttribution()`: Returns an Attribution object by combining:
   - `session_id`: from localStorage `tj_session_id` (same key as trackingService). Read directly from localStorage rather than importing trackingService to avoid circular dependency. If not found, generate via crypto.randomUUID() and persist (same logic as trackingService.getSessionId).
   - `page_path`: `window.location.pathname`
   - `referrer`: `document.referrer || ''`
   - `utm_source/medium/campaign`: from sessionStorage `tj_utm` (parsed JSON), falling back to undefined
   - `device_type`: from getDeviceType()

Import: Only standard Web APIs. No Supabase import needed.
