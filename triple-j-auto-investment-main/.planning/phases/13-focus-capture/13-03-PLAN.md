<task type="auto">
  <name>Task 2: Remove SplashScreen for immediate content visibility</name>
  <files>triple-j-auto-investment-main/App.tsx</files>
  <action>
Remove the SplashScreen wrapper from the App component to eliminate the 1.7-second delay before content is visible. The hero animation IS the new first impression -- no separate splash needed.

**In the App() function** (around lines 625-639):

Current code:
```tsx
export default function App() {
  return (
    <LanguageProvider>
      <StoreProvider>
        <Router>
          <ScrollToTop />
          <HashRedirect />
          <SplashScreen duration={1200}>
            <AppContent />
          </SplashScreen>
        </Router>
      </StoreProvider>
    </LanguageProvider>
  );
}
```

Replace with:
```tsx
export default function App() {
  return (
    <LanguageProvider>
      <StoreProvider>
        <Router>
          <ScrollToTop />
          <HashRedirect />
          <AppContent />
        </Router>
      </StoreProvider>
    </LanguageProvider>
  );
}
```

**Also remove the SplashScreen import** at the top of App.tsx (around line 9):
Remove: `import { SplashScreen } from './components/SplashScreen';`

**Do NOT delete the SplashScreen component file** (components/SplashScreen.tsx). It may be useful in the future or as reference. Just remove its usage from App.tsx.

**Why remove entirely instead of shortening:**
The SplashScreen uses sessionStorage to skip on repeat visits, but FIRST-TIME visitors (the exact target audience for Phase 13's "3-second first impression") ALWAYS see it. Even reducing to 500ms still means 500ms of black screen with a crest logo before the hero animation. The hero animation itself is the new "splash" -- the abstract SVG paths animating in over 1-2 seconds serve the same brand-establishment purpose that the splash screen served, but without blocking content.

**What about the content fade-in that SplashScreen provided?**
The SplashScreen's `isComplete` state controlled opacity/transform on the children wrapper. With SplashScreen removed, AppContent renders immediately at full opacity. The hero section in Home.tsx already has its own Framer Motion entrance animations (staggered from delay 0.3 to 1.1), so the content still has a polished entrance -- just driven by the hero's own animations instead of a wrapper.
