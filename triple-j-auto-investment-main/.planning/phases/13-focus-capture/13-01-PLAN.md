  <action>
Enhance the existing `useEffect` in `LanguageProvider` to check `navigator.language` when no saved language preference exists in localStorage.

Current code (lines 16-21):
```tsx
useEffect(() => {
    const savedLang = localStorage.getItem('tj_lang') as Language;
    if (savedLang && (savedLang === 'en' || savedLang === 'es')) {
        setLangState(savedLang);
    }
}, []);
```

Replace with:
```tsx
useEffect(() => {
    const savedLang = localStorage.getItem('tj_lang') as Language;
    if (savedLang && (savedLang === 'en' || savedLang === 'es')) {
        setLangState(savedLang);
        return;
    }
    // Auto-detect from browser language for first-time visitors
    const browserLang = navigator.language || (navigator.languages && navigator.languages[0]) || 'en';
    if (browserLang.startsWith('es')) {
        setLangState('es');
        localStorage.setItem('tj_lang', 'es');
    }
    // English is default -- no action needed for non-Spanish browsers
}, []);
```

This is a 4-line addition. The logic:
1. If localStorage has a saved preference, use it (existing behavior, unchanged)
2. If NO saved preference (first visit), check navigator.language
3. If browser language starts with 'es' (es, es-MX, es-419, etc.), set Spanish and save to localStorage
4. Otherwise, default to English (no action needed since state initializes as 'en')
