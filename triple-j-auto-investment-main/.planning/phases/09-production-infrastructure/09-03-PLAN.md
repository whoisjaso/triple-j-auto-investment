<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Create storage buckets with RLS, configure app.settings, activate plate cron</name>
  <action>
    **Create storage buckets:**
    Go to Supabase Dashboard -> Storage -> New Bucket and create each:

    1. `rental-agreements` -- Private (toggle OFF "Public bucket")
    2. `rental-photos` -- Public (toggle ON "Public bucket")
    3. `plate-photos` -- Public (toggle ON "Public bucket")
    4. `insurance-cards` -- Public (toggle ON "Public bucket")

    **Add RLS policies for storage:**
    Go to SQL Editor and run this entire block:

    ```sql
    -- Admin can upload files to all buckets
    CREATE POLICY "Admins can upload files"
    ON storage.objects FOR INSERT TO authenticated
    WITH CHECK (
      auth.uid() IN (SELECT id FROM public.profiles WHERE is_admin = true)
    );

    -- Admin can view all files
    CREATE POLICY "Admins can view files"
    ON storage.objects FOR SELECT TO authenticated
    USING (
      auth.uid() IN (SELECT id FROM public.profiles WHERE is_admin = true)
    );

    -- Public can view files in public buckets
    CREATE POLICY "Public can view public bucket files"
    ON storage.objects FOR SELECT TO anon
    USING (bucket_id IN ('rental-photos', 'plate-photos', 'insurance-cards'));

    -- Admin can delete files
    CREATE POLICY "Admins can delete files"
    ON storage.objects FOR DELETE TO authenticated
    USING (
      auth.uid() IN (SELECT id FROM public.profiles WHERE is_admin = true)
    );

    -- Admin can update (overwrite) files
    CREATE POLICY "Admins can update files"
    ON storage.objects FOR UPDATE TO authenticated
    USING (
      auth.uid() IN (SELECT id FROM public.profiles WHERE is_admin = true)
    );
    ```

    **Configure app.settings for pg_cron:**
    The cron jobs need to know the Supabase URL and service role key to call Edge Functions. Run in SQL Editor:

    ```sql
    -- Set database-level settings for cron jobs
    ALTER DATABASE postgres SET app.settings.supabase_url = 'https://YOUR_PROJECT_REF.supabase.co';
    ALTER DATABASE postgres SET app.settings.service_role_key = 'YOUR_SERVICE_ROLE_KEY';
    ```

    Replace `YOUR_PROJECT_REF` and `YOUR_SERVICE_ROLE_KEY` with your actual values from Plan 02.

    **IMPORTANT:** After running the ALTER DATABASE commands, you need to reconnect for them to take effect. Close the SQL Editor tab and reopen it, or refresh the page.

    **Activate check-plate-alerts cron (commented out in migration 07):**
    Run in SQL Editor:

    ```sql
    SELECT cron.schedule(
      'check-plate-alerts',
      '*/30 * * * *',
      $$
      SELECT net.http_post(
        url := current_setting('app.settings.supabase_url') || '/functions/v1/check-plate-alerts',
        headers := jsonb_build_object(
          'Authorization', 'Bearer ' || current_setting('app.settings.service_role_key'),
          'Content-Type', 'application/json'
        ),
        body := '{}'::jsonb
      );
      $$
    );
    ```

    **Verify everything:**
    Run in SQL Editor:

    ```sql
    -- Verify storage buckets
    SELECT id, name, public FROM storage.buckets ORDER BY name;
    -- Should show 4 buckets

    -- Verify cron jobs (both should be active)
    SELECT jobname, schedule, active FROM cron.job ORDER BY jobname;
    -- Should show: check-plate-alerts (*/30 * * * *) and process-notification-queue (* * * * *)

    -- Wait 2 minutes, then check if cron is succeeding
    SELECT jobname, status, return_message, start_time
    FROM cron.job_run_details
    ORDER BY start_time DESC
    LIMIT 5;
    -- Status should be 'succeeded' (if Edge Functions are deployed and secrets are set)
    ```
