**Remove static title and meta description (React 19 manages these per-page now):**

Remove these lines from `<head>`:
- Line 7: `<title>Triple J Auto Investment | Pre-Owned Cars Houston | Sales &amp; Rentals</title>`
- Lines 8-9: `<meta name="description" content="Triple J Auto Investment: Trusted pre-owned vehicles for Houston families. Sales and rentals in the $3,000-$8,000 range. ..." />`

WHY: Plan 01 adds per-page `<title>` and `<meta name="description">` via React 19's native hoisting. If the static tags remain in index.html, there will be duplicate tags in the DOM. React 19 does NOT remove pre-existing static HTML meta tags -- it appends new ones. Browsers use the first `<title>` encountered, so the static one would always win, defeating per-page titles.

KEEP these in index.html (they are NOT managed per-page):
- `<meta name="keywords">` (site-wide keyword targeting, not per-page)
- All OG tags (these will be overridden by React 19 per-page OG tags on navigation, but serve as fallback for the initial HTML load and crawlers that don't execute JS)
- All geo meta tags
- All schema JSON-LD scripts
- Favicon, font, and importmap elements

**Fix AutoDealer schema -- Add LocalBusiness to @type (line 54):**

Change:
```json
"@type": "AutoDealer",
```
To:
```json
"@type": ["AutoDealer", "LocalBusiness"],
```

WHY: The Phase 12 success criteria explicitly require "Schema markup on the homepage accurately reflects LocalBusiness + AutoDealer." AutoDealer is a subtype of LocalBusiness in schema.org, but explicitly declaring both ensures maximum compatibility with Google's structured data processing and the Google Business Profile.
