    **Inventory.tsx (MEDIUM risk):**
    1. **CRITICAL: Fix carousel controls.** The vehicle card image carousel has `opacity-0 group-hover:opacity-100` on prev/next buttons. On touch devices, hover doesn't exist, so users can NEVER see these buttons. Fix by:
       - Making controls always visible on mobile: `opacity-100 md:opacity-0 md:group-hover:opacity-100`
       - OR adding a touch/swipe gesture using onTouchStart/onTouchEnd to detect swipe direction
       - The simplest fix: make controls always visible on mobile with a semi-transparent background
    2. Verify filter bar (dropdowns + search) doesn't overflow at 375px. If three dropdowns sit in a row, they may need to stack or scroll horizontally.
    3. Verify the vehicle detail modal works at 375px (research says it's already fullscreen on mobile -- confirm).
    4. Check that the modal image gallery is usable on mobile -- swipe or tap to navigate.

    **VinLookup.tsx (HIGH risk):**
    1. The three-column grid (`grid-cols-1 lg:grid-cols-3`) should collapse to single column at 375px. Verify.
    2. The terminal-style UI input area: verify it doesn't overflow at 375px. If it has fixed widths or monospace text that's too wide, add `overflow-x-auto` or reduce font size on mobile.
    3. Results grid `grid-cols-2 md:grid-cols-4`: two columns at 375px means each column is ~170px minus padding. Verify text doesn't overflow. If it does, switch to `grid-cols-1 sm:grid-cols-2` for narrower viewports.
    4. Ensure all text is readable (no text-[9px] or similar extremely small text that's unreadable on mobile).

    **App.tsx -- Navbar and Footer touch targets:**
    1. **Footer social icons:** Currently `<Facebook size={20} />` and `<Twitter size={20} />` with only hover classes, no padding. Add `p-3` (or `min-h-[44px] min-w-[44px] flex items-center justify-center`) to make tap targets 44x44px. This is the simplest fix -- just add padding to the `<a>` wrapper.
    2. **Mobile language toggle in navbar:** Currently `p-2` with `text-[10px]`. Verify the touch target meets 44px. The mobile trigger area already looks adequate (`p-3`), but the language toggle next to it (`p-2` + small text) may be too small. Increase to `p-3` or add `min-h-[44px] min-w-[44px]`.
    3. **Footer legal links:** Currently `py-1` which is too short for tap targets. Change to `py-3` to meet 44px touch target height.
    4. **Desktop language toggle:** The `text-[9px]` toggle with `<Globe size={12} />` is tiny. On desktop this is fine (mouse precision), but verify it's not rendered on mobile. If it IS visible on mobile, increase touch target size. (Check if `hidden md:flex` is on the desktop nav section.)
    5. **Mobile menu links:** Currently `py-4` or `py-2` -- ensure all are at least `py-3` for adequate vertical touch targets.

    **Important constraints:**
    - Do NOT change the visual design, colors, or layout philosophy -- only fix sizing, overflow, and touch targets
    - Do NOT modify admin page routing or admin components
    - Keep all animations as-is per CONTEXT
    - If a responsive class already exists and is correct, don't change it
    - Add `overflow-x-hidden` to page wrappers ONLY if needed (don't add it globally -- it can break sticky positioning)
