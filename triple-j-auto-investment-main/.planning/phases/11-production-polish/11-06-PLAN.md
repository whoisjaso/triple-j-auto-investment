  <name>Task 1: Run production build and automated verification checks</name>

  <action>
    Run the following automated verification steps BEFORE asking the user to do visual verification:

    1. **Production build:** Run `npm run build` in the triple-j-auto-investment-main directory. Confirm it succeeds with zero errors.

    2. **Console stripping verification:** Examine the production build output (dist/ directory) for any remaining console.log statements. Search the built JS files:
       ```
       grep -r "console\." dist/assets/*.js | head -20
       ```
       Should return zero results (esbuild.drop strips them).

    3. **Alt text audit:** Search all page and component files for `<img` tags without alt attributes:
       ```
       grep -rn '<img' pages/ components/ | grep -v 'alt='
       ```
       Should return zero results.

    4. **Contrast audit:** Search for remaining text-gray-600 on readable text:
       ```
       grep -rn 'text-gray-600' pages/ components/ App.tsx
       ```
       Review any remaining instances -- they should only be on truly decorative/non-essential text.

    5. **Skip-to-content:** Verify App.tsx has the skip link and main has id="main-content".

    6. **Focus indicators:** Search for `focus:outline-none` without a paired `focus:ring`:
       ```
       grep -rn 'focus:outline-none' pages/ components/ App.tsx
       ```
       Each instance should have an accompanying focus:ring or equivalent.

    7. **Translation completeness:** Verify the polish section exists in both en and es objects in translations.ts.

    8. **Start the dev server** for the user to do visual testing:
       ```
       npm run dev
       ```

    Report all findings to the user before proceeding to the checkpoint.
