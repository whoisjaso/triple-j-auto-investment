    **2. Inventory modal focus management (Inventory.tsx):**

    The vehicle detail modal (opened when clicking a vehicle card) needs focus management:
    - When modal opens: move focus to the modal (first focusable element, or the modal container itself with `tabIndex={-1}`)
    - When modal is open: trap Tab/Shift+Tab within the modal (don't let focus escape to background content)
    - When modal closes: return focus to the element that triggered the modal (the vehicle card)
    - Add `Escape` key handler to close the modal
    - Add `role="dialog"`, `aria-modal="true"`, and `aria-label` to the modal container

    Focus trap implementation (simple approach):
    ```tsx
    // Inside modal component
    const modalRef = useRef<HTMLDivElement>(null);

    useEffect(() => {
      if (isOpen && modalRef.current) {
        modalRef.current.focus();

        const handleKeyDown = (e: KeyboardEvent) => {
          if (e.key === 'Escape') {
            onClose();
            return;
          }
          if (e.key === 'Tab') {
            const focusableElements = modalRef.current?.querySelectorAll(
              'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
            );
            if (!focusableElements?.length) return;
            const first = focusableElements[0] as HTMLElement;
            const last = focusableElements[focusableElements.length - 1] as HTMLElement;
            if (e.shiftKey && document.activeElement === first) {
              e.preventDefault();
              last.focus();
            } else if (!e.shiftKey && document.activeElement === last) {
              e.preventDefault();
              first.focus();
            }
          }
        };

        document.addEventListener('keydown', handleKeyDown);
        return () => document.removeEventListener('keydown', handleKeyDown);
      }
    }, [isOpen, onClose]);
    ```

    Also add `aria-hidden="true"` to the background overlay/backdrop of the modal.
