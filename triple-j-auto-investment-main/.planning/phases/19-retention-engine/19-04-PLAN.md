Create the Edge Function following the exact pattern of process-follow-up-queue/index.ts from Phase 18.

**Structure:**
- Import createClient from 'npm:@supabase/supabase-js@2'.
- Import sendSms from '../_shared/twilio.ts'.
- Import sendEmail from '../_shared/resend.ts'.

**Deno.serve handler:**
1. Create Supabase service role client using Deno.env.get('SUPABASE_URL') and Deno.env.get('SUPABASE_SERVICE_ROLE_KEY').

2. Query review_requests: SELECT *, registrations(customer_name, customer_phone, customer_email, vehicle_year, vehicle_make, vehicle_model) WHERE sent = false AND send_after <= NOW(), ORDER BY send_after ASC, LIMIT 50.

3. Define GOOGLE_REVIEW_LINK constant with a TODO placeholder: `const GOOGLE_REVIEW_LINK = 'https://g.page/r/YOUR_PLACE_ID/review'; // TODO(business-data): Replace with actual Triple J Google review link`.

4. For each queue item, in a try/catch:
   - Extract registration data from the joined registrations object.
   - Extract firstName from customer_name (split on space, take first).

   **SMS channel (if channel === 'sms' AND customer_phone exists):**
   - Initial request_type: "Hi {firstName}! You've been driving your {year} {make} {model} for a few days now. Would you help other Houston families find a trustworthy dealer? Leave a quick Google review here: {GOOGLE_REVIEW_LINK} -- Triple J Auto Investment (832) 400-9760"
   - Followup request_type: "Hi {firstName}, just a gentle reminder -- your experience with your {year} {make} {model} could help another family make a confident decision. Review here: {GOOGLE_REVIEW_LINK} -- Triple J Auto Investment"
   - Call sendSms(customer_phone, smsBody).

   **Email channel (if channel === 'email' AND customer_email exists):**
   - Subject (initial): "How is your {year} {make}?"
   - Subject (followup): "Your experience could help a family"
   - Build HTML email body using buildReviewEmailHtml function (defined below).
   - Call sendEmail({ to: customer_email, subject, html }).

   **On success or error:** Update review_requests SET sent = true, sent_at = NOW() WHERE id = item.id. Mark sent=true even on error to prevent infinite retry loops (Phase 18 established pattern).

5. Return JSON response: { processed: count }.

**buildReviewEmailHtml function:**
- Takes firstName, vehicle info (year, make, model), request_type ('initial' | 'followup'), GOOGLE_REVIEW_LINK.
- Returns inline HTML (no external CSS) with:
  - Header: Triple J Auto Investment (gold text on dark bg).
  - Body: Community-framed message.
  - Initial: "Hi {firstName}, you've been driving your {year} {make} {model} for a few days now. We hope you love it! Other Houston families are looking for a dealer they can trust. Your honest experience -- good or constructive -- helps them make a confident decision. [Leave a Google Review] button linking to GOOGLE_REVIEW_LINK."
  - Followup: "Hi {firstName}, just a gentle reminder -- one review from you could help another family find reliable transportation. We appreciate your honest feedback. [Share Your Experience] button."
  - Footer: Triple J Auto Investment | 8774 Almeda Genoa, Houston TX 77075 | (832) 400-9760.
- Style: inline CSS only. Background: #0a0a0a. Text: #d4d4d4. Gold accent: #d4af37. Button: background #d4af37, color #000, padding 12px 24px, text-decoration none, border-radius 4px.

**CORS headers (if applicable):**
- This function will be invoked by pg_cron via Supabase's internal Edge Function trigger, so CORS is not required. But add basic CORS headers for manual testing: Access-Control-Allow-Origin: *, OPTIONS method handling.

**Error handling per Phase 18 pattern:**
- Per-item try/catch (one failure does not block others).
- Mark sent=true on error to prevent retry loops.
- Twilio error 21610 (opted out): detect by error message string match, mark as sent with no retry.
