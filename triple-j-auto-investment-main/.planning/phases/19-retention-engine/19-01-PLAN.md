Create the Phase 19 database migration with these sections in order:

**Section 1 -- Add mileage_at_purchase to registrations:**
- ALTER TABLE public.registrations ADD COLUMN IF NOT EXISTS mileage_at_purchase INTEGER;
- Add a trigger that sets mileage_at_purchase from the linked vehicles.mileage when current_stage transitions to 'sticker_delivered' (if mileage_at_purchase is still NULL). This ensures the value tracker has accurate starting mileage.

**Section 2 -- owner_referrals table:**
- id UUID PK default uuid_generate_v4()
- registration_id UUID NOT NULL REFERENCES registrations(id) ON DELETE CASCADE
- customer_phone TEXT NOT NULL
- referrer_name TEXT NOT NULL
- referral_code TEXT UNIQUE NOT NULL
- referral_link TEXT NOT NULL (computed as base URL + '/refer/' + code)
- referral_count INTEGER DEFAULT 0
- converted_count INTEGER DEFAULT 0
- reward_tier INTEGER DEFAULT 0 (0=none, 1=$50, 3=$100, 5=$200)
- created_at TIMESTAMPTZ DEFAULT NOW()
- Enable RLS. Policy: authenticated users SELECT only WHERE customer_phone = auth.jwt()->>'phone'. Service role full access.

**Section 3 -- referral_clicks table:**
- id UUID PK default uuid_generate_v4()
- referral_code TEXT NOT NULL (FK to owner_referrals.referral_code ON DELETE CASCADE)
- clicked_at TIMESTAMPTZ DEFAULT NOW()
- device_type TEXT
- Enable RLS. Policy: anon INSERT WITH CHECK (true). Service role SELECT for admin.

**Section 4 -- review_requests table:**
- id UUID PK default uuid_generate_v4()
- registration_id UUID NOT NULL REFERENCES registrations(id) ON DELETE CASCADE
- channel TEXT NOT NULL CHECK (channel IN ('sms', 'email'))
- request_type TEXT NOT NULL DEFAULT 'initial' CHECK (request_type IN ('initial', 'followup'))
- send_after TIMESTAMPTZ NOT NULL
- sent BOOLEAN DEFAULT false
- sent_at TIMESTAMPTZ
- created_at TIMESTAMPTZ DEFAULT NOW()
- UNIQUE (registration_id, channel, request_type)
- Enable RLS. Service role full access only (no direct customer access).

**Section 5 -- Add review_completed to registrations:**
- ALTER TABLE public.registrations ADD COLUMN IF NOT EXISTS review_completed BOOLEAN DEFAULT false;
- This allows the 7-day follow-up to be skipped if the owner marks "I left a review" in the portal.

**Section 6 -- generate_referral_code() function:**
- Takes customer_name TEXT, returns TEXT.
- Pattern: first 3 uppercase letters of name + '-' + 4 random uppercase alphanumeric chars (excluding confusable chars 0/O/I/1).
- Uses LPAD for short names. Uses gen_random_bytes for randomness.

**Section 7 -- auto_create_owner_referral() trigger function:**
- Fires AFTER UPDATE on registrations WHERE NEW.current_stage = 'sticker_delivered' AND OLD.current_stage != 'sticker_delivered'.
- Creates a row in owner_referrals with generated referral_code and referral_link.
- Also snapshots mileage_at_purchase if NULL (from vehicles table via vehicle_id).
- ON CONFLICT DO NOTHING (idempotent).

**Section 8 -- enqueue_review_requests() SECURITY DEFINER function:**
- Finds registrations with current_stage = 'sticker_delivered' AND purchase_date between NOW() - INTERVAL '3 days 2 hours' AND NOW() - INTERVAL '3 days'.
- Filters out registrations that already have an 'initial' review_request.
- CROSS JOIN unnest(ARRAY['sms', 'email']) for dual-channel.
- INSERT with ON CONFLICT DO NOTHING.

**Section 9 -- enqueue_followup_review_requests() SECURITY DEFINER function:**
- Finds registrations with current_stage = 'sticker_delivered' AND purchase_date between NOW() - INTERVAL '7 days 2 hours' AND NOW() - INTERVAL '7 days'.
- Filters out registrations WHERE review_completed = true.
- Filters out registrations that already have a 'followup' review_request.
- CROSS JOIN unnest(ARRAY['sms', 'email']) for dual-channel.
- INSERT with ON CONFLICT DO NOTHING.

**Section 10 -- pg_cron schedules (DO block with EXCEPTION WHEN OTHERS pattern):**
- Schedule enqueue_review_requests() every hour: '0 * * * *'
- Schedule enqueue_followup_review_requests() every hour: '0 * * * *'
- Wrap in DO block with EXCEPTION WHEN OTHERS THEN RAISE NOTICE (Free plan graceful degradation, same pattern as Phase 18).

**Section 11 -- Indexes:**
- CREATE INDEX IF NOT EXISTS idx_owner_referrals_phone ON owner_referrals(customer_phone);
- CREATE INDEX IF NOT EXISTS idx_owner_referrals_code ON owner_referrals(referral_code);
- CREATE INDEX IF NOT EXISTS idx_review_requests_unsent ON review_requests(sent, send_after) WHERE sent = false;
- CREATE INDEX IF NOT EXISTS idx_referral_clicks_code ON referral_clicks(referral_code);
