  <name>Task 1: Fix viewport meta tag (POLISH-02)</name>
  <files>triple-j-auto-investment-main/index.html</files>
  <action>
    On line 6 of index.html, the viewport meta tag currently reads:

    ```
    <meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no, viewport-fit=cover" />
    ```

    Remove `maximum-scale=1.0` and `user-scalable=no` from the content attribute. Keep `viewport-fit=cover` (needed for iPhone notch handling). The result should be:

    ```
    <meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover" />
    ```

    **Why:** `user-scalable=no` and `maximum-scale=1.0` prevent pinch-to-zoom, violating WCAG 1.4.4 (Resize Text). Users with low vision rely on zoom. This is a BLOCKER for accessibility compliance.

    **Do NOT change** any other meta tags, schema markup, or content in index.html.
  </action>
  <verify>
    Read index.html line 6 and confirm:
    - No `user-scalable=no` in the viewport meta
    - No `maximum-scale=1.0` in the viewport meta
    - `width=device-width` and `initial-scale=1.0` still present
    - `viewport-fit=cover` still present
  </verify>

    ```tsx

    ```
    to:
    ```tsx
    <SplashScreen duration={1200}>
    ```

    **In SplashScreen.tsx:**
    1. Change the fade-out duration from 800ms to 500ms. On line 29-31, change:
       ```tsx
       const fadeTimer = setTimeout(() => {
         setPhase('complete');
         sessionStorage.setItem('splashShown', 'true');
       }, duration + 800);
       ```
       to:
       ```tsx
       const fadeTimer = setTimeout(() => {
         setPhase('complete');
         sessionStorage.setItem('splashShown', 'true');

       - Line 102: `transition: 'opacity 0.8s ..., transform 0.8s ...'` -> `transition: 'opacity 0.5s ..., transform 0.5s ...'`

       ```
