    **VehicleJsonLd.tsx** -- Create NEW file.

    Props: `{ vehicle: Vehicle }` (import Vehicle from types.ts)
    - Outputs a `<script type="application/ld+json">` tag with Schema.org/Car structured data
    - Uses dangerouslySetInnerHTML (same pattern as existing schema in index.html)

    JSON-LD structure (from the research document):
    ```json
    {
      "@context": "https://schema.org/",
      "@type": "Car",
      "name": "{year} {make} {model}",
      "description": "{identityHeadline or description}",
      "brand": { "@type": "Brand", "name": "{make}" },
      "model": "{model}",
      "vehicleModelDate": "{year}",
      "itemCondition": "https://schema.org/UsedCondition",
      "mileageFromOdometer": {
        "@type": "QuantitativeValue",
        "value": "{mileage}",
        "unitCode": "SMI"
      },
      "vehicleIdentificationNumber": "{vin}",
      "color": (omit if not available),
      "offers": {
        "@type": "Offer",
        "availability": "https://schema.org/InStock" or "SoldOut" based on status,
        "price": "{price}",
        "priceCurrency": "USD",
        "seller": {
          "@type": "AutoDealer",
          "name": "Triple J Auto Investment",
          "telephone": "(832) 400-9760",
          "address": {
            "@type": "PostalAddress",
            "streetAddress": "8774 Almeda Genoa Rd",
            "addressLocality": "Houston",
            "addressRegion": "TX",
            "postalCode": "77075"
          }
        }
      },
      "image": "{imageUrl}",
      "url": "https://triplejautoinvestment.com/vehicles/{slug}"
    }
    ```

    Only include fields that have values (don't output null/undefined fields in JSON-LD).
    This component renders no visible UI -- only the script tag for search engines.
