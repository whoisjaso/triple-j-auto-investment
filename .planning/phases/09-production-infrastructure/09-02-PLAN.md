       - `pg_cron` (required by migration 04 for notification scheduling)
       - `pg_net` (required by migration 04 for HTTP calls from cron)
    3. Verify all 4 show as "Enabled"

    Go to SQL Editor and run each file one at a time in this exact order. After each, verify it completes without errors before proceeding to the next.

    ```
    triple-j-auto-investment-main/supabase/schema.sql
    triple-j-auto-investment-main/supabase/registration_ledger.sql
    triple-j-auto-investment-main/supabase/migrations/02_registration_schema_update.sql
    triple-j-auto-investment-main/supabase/migrations/03_customer_portal_access.sql

    triple-j-auto-investment-main/supabase/migrations/05_registration_checker.sql
    triple-j-auto-investment-main/supabase/migrations/06_rental_schema.sql
    triple-j-auto-investment-main/supabase/migrations/07_plate_tracking.sql
    triple-j-auto-investment-main/supabase/migrations/08_rental_insurance.sql
    ```

    For each migration:
    1. Open the file in your code editor
    2. Copy the entire contents
    3. Paste into Supabase SQL Editor
    4. Click "Run"
    5. Verify "Success" with no errors
    6. Proceed to the next file

    WHERE extname IN ('uuid-ossp', 'btree_gist', 'pg_cron', 'pg_net');
