<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Export vehicles from dev Supabase and import to production</name>
  <action>
    **First: Check vehicle image URLs in dev database**

    Before migrating, you need to know where vehicle gallery images are hosted. Go to the DEV Supabase project SQL Editor and run:

    ```sql
    -- Check a sample of vehicle gallery URLs to see where images are hosted
    SELECT id, make, model, gallery->0 as first_image_url
    FROM vehicles
    WHERE gallery IS NOT NULL AND jsonb_array_length(gallery) > 0
    LIMIT 5;
    ```

    Look at the URLs:
    - If they start with `https://scgmpliwlfabnpygvbsy.supabase.co/storage/` -- images are in dev Supabase storage and need special handling (see note below)
    - If they start with `https://` pointing to external sites (Imgur, car listing sites, etc.) -- they will continue working after migration
    - If they are relative paths or missing -- no action needed

    **If images are in dev Supabase storage:**
    The dev Supabase project will eventually be deleted, so these images need to be downloaded and re-uploaded. This is complex enough to warrant its own approach:
    1. Download all images from dev storage
    2. Upload to production storage or an external image host
    3. Update the gallery URLs in the migrated data
    Report this finding and we will handle it as a gap closure if needed.

    **Export vehicles data from dev Supabase:**

    Option A -- SQL Editor copy (simplest for small datasets):
    1. Go to DEV Supabase SQL Editor
    2. Run:
    ```sql
    SELECT * FROM vehicles ORDER BY created_at;
    ```
    3. Click "Download CSV" to export the data
    4. Go to PRODUCTION Supabase -> Table Editor -> vehicles
    5. Click "Insert rows" -> "Import data from CSV"
    6. Upload the CSV file
    7. Map columns and import

    Option B -- pg_dump (more reliable for larger datasets):
    If you have `psql`/`pg_dump` available:
    ```bash
    # Export from dev (get connection string from dev Supabase -> Settings -> Database -> Connection string -> URI)
    pg_dump \
      --data-only \
      --table=public.vehicles \
      --column-inserts \
      "postgresql://postgres.[DEV_REF]:[DEV_PASSWORD]@aws-0-us-east-2.pooler.supabase.com:6543/postgres" \
      > vehicles_data.sql

    # Import to production
    psql \
      "postgresql://postgres.[PROD_REF]:[PROD_PASSWORD]@aws-0-us-east-1.pooler.supabase.com:6543/postgres" \
      < vehicles_data.sql
    ```

    Option C -- Manual entry (if only a few vehicles):
    If the inventory is small (< 10 vehicles), it may be faster to re-enter them through the admin UI on the production site.

    **After import, verify:**
    Go to PRODUCTION Supabase SQL Editor:
    ```sql
    SELECT id, make, model, year, price, status FROM vehicles ORDER BY created_at;
    ```
    Confirm all vehicles are present.

    **Then verify on the live site:**
    1. Go to https://triplejautoinvestment.com
    2. Navigate to the inventory page
    3. Verify vehicles appear with correct details
    4. Click on a vehicle to verify gallery images load
  </action>
