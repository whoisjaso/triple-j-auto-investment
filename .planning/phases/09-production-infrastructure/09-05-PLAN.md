  <action>
    **Set environment variables in Vercel:**
    1. Go to https://vercel.com -> Your project (triple-j-auto-investment or similar)
    2. Settings -> Environment Variables
    3. Add each variable for the "Production" environment:

    | Variable | Value | Source |
    |----------|-------|--------|
    | VITE_SUPABASE_URL | `https://YOUR_PROJECT_REF.supabase.co` | Supabase Dashboard -> Settings -> API |
    | VITE_SUPABASE_ANON_KEY | Your anon public key | Supabase Dashboard -> Settings -> API |
    | VITE_GEMINI_API_KEY | Your Gemini API key | Google AI Studio (if you have one; if not, leave empty and AI descriptions will gracefully degrade) |
    | VITE_RETELL_API_KEY | Your NEW Retell API key | Retell AI Dashboard -> API Keys (rotate the key first if using the exposed one) |
    | VITE_RETELL_OUTBOUND_AGENT_ID | Your agent ID | Retell AI Dashboard -> Agents |
    | VITE_RETELL_OUTBOUND_NUMBER | Your Retell outbound number | Retell AI Dashboard -> Phone Numbers |

    **DO NOT add these (they are legacy and should not exist in production):**
    - VITE_ADMIN_EMAIL
    - VITE_ADMIN_PASSWORD
    - Any VITE_EMAILJS_* variables (unless you plan to use EmailJS)

    4. After adding all 6 variables, verify they are shown in the Environment Variables list

    **Retell API key rotation:**
    Before adding the Retell API key to Vercel, check if the old key `key_86a41f8fad555c03e879a71d0398` is still in use:
    1. Go to Retell AI Dashboard -> API Keys
    2. If the old key is listed, create a new API key
    3. Delete the old key (it was exposed in the public repo)
    4. Use the NEW key for the Vercel env var
    5. Update the agent configuration if needed

    **Trigger deployment:**

    **Wait for build to complete:**
    - Vercel Dashboard -> Deployments should show a new deployment building
    - Build typically takes 1-3 minutes for a Vite project
    - Check build logs for any errors

    **Verify the deployment:**
    1. Click the deployment URL (something like `triple-j-auto-investment-xxx.vercel.app`)
    2. Open browser DevTools (F12) -> Console
    3. Verify NO errors about missing environment variables

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Connect custom domain at Namecheap and Vercel</name>
  <action>
    **Add domain in Vercel:**
    1. Go to Vercel -> Your project -> Settings -> Domains
    2. Add domain: `triplejautoinvestment.com`
    3. Also add: `www.triplejautoinvestment.com`
    4. Vercel will show the DNS records you need to add

    **Configure DNS at Namecheap:**
    1. Go to Namecheap -> Domain List -> triplejautoinvestment.com -> Advanced DNS
    2. Add/update these records:
       - **A Record:** Host = `@`, Value = `76.76.21.21` (Vercel IP), TTL = Automatic
       - **CNAME Record:** Host = `www`, Value = `cname.vercel-dns.com`, TTL = Automatic
    3. If there are existing A records pointing elsewhere, update them to the Vercel IP
    4. Keep the Resend DNS records you added in Plan 01 (SPF + DKIM) -- do NOT remove those

    **Wait for DNS propagation:**
    - Can take 5 minutes to 48 hours (typically 15-30 minutes with Namecheap)
    - Check status in Vercel -> Domains -- it will show a checkmark when verified
    - Vercel automatically provisions an SSL certificate once DNS resolves

    **Verify:**
    1. Visit `https://triplejautoinvestment.com` in your browser
    2. Check that the SSL padlock icon is present (valid certificate)
    3. Visit `https://www.triplejautoinvestment.com` -- should redirect to the non-www version (or vice versa, depending on Vercel config)
    4. Open DevTools -> Console to verify no errors
  </action>
  <verify>
    - `https://triplejautoinvestment.com` loads the site with valid SSL
    - `https://www.triplejautoinvestment.com` redirects properly
    - Vercel Domains page shows both domains with green checkmarks
    - No certificate warnings in the browser
  </verify>
