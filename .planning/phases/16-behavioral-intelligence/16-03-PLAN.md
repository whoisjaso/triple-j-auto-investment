**useRecentlyViewed.ts:**
Follow the exact pattern from useSavedVehicles.ts (localStorage + useState + cross-tab sync).

Constants:
- `STORAGE_KEY = 'tj_recently_viewed'`
- `MAX_RECENT = 8`

State shape: Array of `{ vehicleId: string; viewedAt: string }` objects, most recent first.

Functions:
- `addViewed(vehicleId: string)`: Remove existing entry for this vehicleId (dedup), prepend new entry with current ISO timestamp, slice to MAX_RECENT, persist to localStorage.
- Read from localStorage on init (with try/catch for parse errors).
- Cross-tab sync via `window.addEventListener('storage', ...)` for the key (same pattern as useSavedVehicles).

Return: `{ entries, addViewed, vehicleIds: entries.map(e => e.vehicleId) }`.

**recommendationService.ts:**
Create a pure function module with no Supabase dependency.

`scoreSimilarity(a: Vehicle, b: Vehicle): number`:
- Same make: +3
- Price within 20%: +2
- Year within 3: +1
- Mileage within 20,000: +1
- Maximum score per pair: 7

`getRecommendations(viewedVehicles: Vehicle[], allVehicles: Vehicle[], limit = 4): Vehicle[]`:
- Filter out viewed vehicles (by ID) and non-Available vehicles
- Score each candidate against ALL viewed vehicles (sum scores)
- Sort descending by score, take top `limit`
- Return Vehicle array

Import Vehicle type from '../types'.
