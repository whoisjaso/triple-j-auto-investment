**trackingService.ts:**
Create the event buffering service following the research pattern:

1. `getSessionId()`: Read from localStorage key `tj_session_id`. If missing, generate via `crypto.randomUUID()` and persist. Export this -- attribution capture needs it.

2. `getDeviceType()`: Return 'mobile' (width < 768), 'tablet' (width < 1024), or 'desktop' based on `window.innerWidth`. Matches Tailwind breakpoints.

3. `trackEvent(event: Omit<TrackingEvent, 'session_id' | 'created_at'>)`: Push to in-memory buffer array with session_id and created_at added. If buffer length >= 20, auto-flush. This function must be fire-and-forget (no async, no await). Export it.

4. `flush()`: If buffer is empty, return. Copy buffer, clear it, then `supabase.from('session_events').insert(events)`. On error, re-queue events to front of buffer and console.warn (do NOT throw). Export for testing but primary usage is internal.

5. Periodic flush: `setInterval(flush, 30_000)` at module level.

6. `flushOnUnload()`: Uses `fetch` with `keepalive: true` (NOT sendBeacon -- needs apikey/Authorization headers). Send to `${SUPABASE_URL}/rest/v1/session_events` with Content-Type, apikey, Authorization (Bearer anon key), and Prefer: return=minimal headers. Fire-and-forget with `.catch(() => {})`. Clear buffer after sending.

7. Register listeners at module level:
   - `visibilitychange`: flush when `document.visibilityState === 'hidden'` (primary)
   - `beforeunload`: call flushOnUnload (fallback)

Import supabase from '../supabase/config'. Read VITE_SUPABASE_URL and VITE_SUPABASE_ANON_KEY from import.meta.env for the fetch keepalive call.
