<objective>
Implement honest urgency badges that surface real inventory signals -- Just Arrived, Popular, and Offer Received.

Purpose: Urgency badges create honest scarcity signals that accelerate decision-making. Unlike fake countdown timers or fabricated "X people are viewing this" messages, these badges are strictly data-driven: Just Arrived uses the actual dateAdded, Popular uses actual view counts from vehicle_view_counts, and Offer Received uses actual reserve-level leads. This honesty is a core architectural requirement.

Output: urgencyService.ts (badge computation), useUrgencyBadges hook (data fetching + caching), UrgencyBadge component (rendering), and integration into Inventory cards and VehicleDetail page.
</objective>

**urgencyService.ts:**
Pure logic module -- no Supabase dependency. Import Vehicle from types.

Define badge types:
```typescript
export type UrgencyBadgeType = 'just_arrived' | 'popular' | 'offer_received';

export interface UrgencyBadgeData {
  type: UrgencyBadgeType;
  label: string;
  labelEs: string;
}
```

`computeBadges(vehicle: Vehicle, viewCount: number, hasOffer: boolean): UrgencyBadgeData[]`:
1. Just Arrived: If `vehicle.dateAdded` exists and is within the last 7 days (compare Date.now() - new Date(dateAdded).getTime() to 7 * 86400000), add `{ type: 'just_arrived', label: 'Just Arrived', labelEs: 'Recien Llegado' }`.
2. Popular: If `viewCount >= 10`, add `{ type: 'popular', label: \`Popular \u00b7 ${viewCount} views\`, labelEs: \`Popular \u00b7 ${viewCount} vistas\` }`.
3. Offer Received: If `hasOffer` is true, add `{ type: 'offer_received', label: 'Offer Received', labelEs: 'Oferta Recibida' }`.

Return array of applicable badges (may be empty, may have multiple).

**useUrgencyBadges.ts:**
React hook that fetches the data needed for badge computation and returns a per-vehicle lookup.

On mount (and on a 5-minute refetch interval):
1. Fetch view counts: `supabase.from('vehicle_view_counts').select('vehicle_id, views_7d')`. Build a Map<vehicleId, views_7d>.
2. Fetch offer vehicles: `supabase.from('leads').select('vehicle_id').gte('commitment_level', 3)`. Build a Set<vehicleId> of vehicles with offers.
3. Both queries fire in parallel (Promise.all).

Return: `getBadges(vehicle: Vehicle): UrgencyBadgeData[]` -- a function that components call for each vehicle. Internally calls `computeBadges(vehicle, viewCountMap.get(vehicle.id) || 0, offerSet.has(vehicle.id))`.

Also return `loading: boolean` and `refresh(): void`.

Cache the fetched data in state. The 5-minute interval uses `setInterval` with cleanup in useEffect return. This avoids hammering Supabase while keeping badges reasonably fresh.

Handle errors gracefully: if either query fails, return empty badges for all vehicles (console.warn, do not throw). The site works fine without badges.

Import supabase from '../supabase/config'.
